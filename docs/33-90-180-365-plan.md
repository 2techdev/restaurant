## What Must NOT Happen in 365 Days

1. Starting Germany fiscal before Swiss pilot has 30+ days stable and 5+ customers
2. Building LAN sync — architecture decision is final: cloud sync only
3. Starting online ordering before cloud sync is proven in production
4. Taking on > 5 pilot customers before multi-device cloud sync is working
5. Publishing to Play Production before 30 days stable in closed beta
6. Committing the release keystore to git
7. Deploying cloud backend without HTTPS and automated daily backups
8. Changing SQLite schema without a Drift migration strategy
9. Building custom backoffice features into GastroCore — that is a separate project
10. Losing the pilot restaurant relationship — weekly check-ins are mandatory
