## What Must NOT Happen in 365 Days

1. Starting Germany fiscal before Swiss pilot is stable (minimum 30 days, 5 customers)
2. Starting online ordering before LAN sync is proven in production
3. Adding Redis or microservices before demonstrating PostgreSQL is the bottleneck
4. Taking on > 5 pilot customers before multi-device is working
5. Publishing to Play Production before 30 days stable in closed beta
6. Committing the release keystore to git
7. Deploying cloud backend without HTTPS and automated backups
8. Changing the SQLite schema in a way that breaks existing installations without migration
9. Building custom backoffice features into GastroCore — that is a separate project
10. Losing the pilot restaurant relationship — weekly check-ins are mandatory
