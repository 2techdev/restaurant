2. About GastroCore POS

GastroCore POS is a point-of-sale application designed for restaurants, cafés, and food-service businesses. The app operates offline-first — all core data is stored locally on the device. An optional cloud synchronization feature may be enabled by the user.

3. Data We Collect

3.1 Restaurant Operational Data

The following data is entered and managed by the restaurant operator:

  • Orders and order items (products, quantities, prices, timestamps)
  • Product catalog (names, categories, prices, descriptions)
  • Table and floor plan configurations
  • Employee records (name, role, PIN / access code)
  • Customer receipts and transaction records
  • Daily reports and sales summaries

Note: This data belongs exclusively to the restaurant operator. We do not access, analyse, or monetise this data.

3.2 Device Information

The app collects minimal device information required for operation:

  • Device model and Android version (for compatibility)
  • App version number
  • Unique device identifier (for license management only)

3.3 Data We Do NOT Collect

  • No personal data of restaurant end-customers (diners)
  • No location data
  • No advertising identifiers (AAID)
  • No behavioural tracking or analytics
  • No microphone, camera, or contacts access

4. Data Storage & Retention

4.1 Local Storage (Default)

All operational data is stored locally on the Android device using an encrypted SQLite database. Data remains on the device until manually deleted by the operator or the app is uninstalled.

4.2 Optional Cloud Sync

If the operator enables cloud synchronisation, data is transmitted via encrypted HTTPS connections to secure servers. Operators can disable cloud sync and request deletion of cloud data at any time by contacting developer@2tech.ch.

4.3 Retention Periods

Data TypeRetention
Local operational dataUntil manually deleted or app uninstalled
Cloud sync data (if enabled)Until deletion requested by operator

2. Über GastroCore POS

GastroCore POS ist eine Kassensoftware (Point-of-Sale-Anwendung) für Restaurants, Cafés und Gastronomiebetriebe. Die App arbeitet offline-first – alle Kerndaten werden lokal auf dem Gerät gespeichert. Eine optionale Cloud-Synchronisierung kann vom Betreiber aktiviert werden.

3. Erhobene Daten

3.1 Betriebsdaten des Restaurants

Folgende Daten werden vom Restaurantbetreiber eingegeben und verwaltet:

  • Bestellungen und Bestellpositionen (Produkte, Mengen, Preise, Zeitstempel)
  • Produktkatalog (Namen, Kategorien, Preise, Beschreibungen)
  • Tisch- und Raumplankonfigurationen
  • Mitarbeiterdaten (Name, Rolle, PIN / Zugangscode)
  • Quittungen und Transaktionsbelege
  • Tagesberichte und Verkaufsübersichten

Hinweis: Diese Daten gehören ausschliesslich dem Restaurantbetreiber. Wir greifen auf diese Daten nicht zu, analysieren oder vermarkten sie nicht.

3.2 Geräteinformationen

Die App erhebt minimale Geräteinformationen, die für den Betrieb erforderlich sind:

  • Gerätemodell und Android-Version (für Kompatibilität)
  • App-Versionsnummer
  • Eindeutige Gerätekennung (nur für Lizenzverwaltung)

3.3 Nicht erhobene Daten

  • Keine personenbezogenen Daten der Restaurantgäste
  • Keine Standortdaten
  • Keine Werbe-IDs (AAID)
  • Kein Verhaltens-Tracking oder Analytics
  • Kein Zugriff auf Mikrofon, Kamera oder Kontakte

4. Datenspeicherung & Aufbewahrung

4.1 Lokale Speicherung (Standard)

Alle Betriebsdaten werden lokal auf dem Android-Gerät in einer verschlüsselten SQLite-Datenbank gespeichert. Die Daten verbleiben auf dem Gerät, bis sie vom Betreiber manuell gelöscht werden oder die App deinstalliert wird.

4.2 Optionale Cloud-Synchronisierung

Wenn der Betreiber die Cloud-Synchronisierung aktiviert, werden Daten über verschlüsselte HTTPS-Verbindungen an sichere Server übertragen. Betreiber können die Cloud-Sync jederzeit deaktivieren und die Löschung von Cloud-Daten unter developer@2tech.ch anfordern.

4.3 Aufbewahrungsfristen
